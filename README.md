
# Kuphila AI — Website

## 🩺 Overview

**Kuphila AI** is a health-tech startup dedicated to bridging the healthcare gap in Africa through cutting-edge Artificial Intelligence. This repository contains the source code for the official company website, built with **Angular**.

Our primary focus is the development of accessible, high-accuracy diagnostic solutions, such as **CerviPredict**, which utilizes Computer Vision and Large Multimodal Models (VLMs) like **MedGemma** for automated cervical cancer screening.



## 🚀 Key Features

  * **Responsive Design:** Fully optimized for mobile, tablet, and desktop viewing.
  * **Performance First:** Built using Angular's best practices for fast loading and SEO.
  * **Modern UI/UX:** Clean, medical-grade interface using *Plus Jakarta Sans* and *DM Serif Display* typography.
  * **SEO Optimized:** Integrated Open Graph tags and Structured Data (JSON-LD) for better search engine visibility.



## 💻 Getting Started

### Prerequisites

Ensure you have the following installed on your local machine:

  * [Node.js](https://nodejs.org/) (LTS version)
  * [Angular CLI](https://angular.io/cli) (`npm install -g @angular/cli`)

### Installation

1.  **Clone the repository:**

    ```bash
    git clone https://github.com/KuphilaAI/Kuphila-AI-website.git
    cd Kuphila-AI-website
    ```

2.  **Install dependencies:**

    ```bash
    npm install
    ```

3.  **Run the development server:**

    ```bash
    ng serve
    ```

    Navigate to `http://localhost:4200/`. The app will automatically reload if you change any of the source files.



## 📁 Project Structure

```text
src/
├── app/                # Main application logic and components
├── assets/             # Images, icons (heart-tech.png), and static files
├── environments/       # Configuration for production/development
├── index.html          # Main HTML entry point (SEO & Meta Tags)
└── styles.scss         # Global styles and theme variables
```



## 🧪 Testing & Build

  * **Build for Production:**
    ```bash
    ng build --configuration production
    ```
    The build artifacts will be stored in the `dist/` directory.
  * **Running Unit Tests:**
    ```bash
    ng test
    ```
